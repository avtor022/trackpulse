## Основные принципы

### Архитектура
```
UI → Service → Repository → Database
```
Никогда не нарушайте это направление зависимостей!

### Локализация
Все тексты должны использоваться через `locale.T("key")`.

### Обновление UI
Из горутин обновляйте UI только через `fyne.Do(func() { ... })`.

### LapService
LapService обрабатывает RFID-сканирования асинхронно с использованием:
- Буферизированного канала на 200 сканирований
- Фонового worker для пакетной записи в БД
- Кэшей транспондеров и участников для O(1) поиска
- **Метод `markTransponderWorked(participantID)`** — автоматически устанавливает флаг `transponder_worked = true` при первом проезде через антенну

### Отображение регистрации кругов
Панель мониторинга отображает таблицу участников сразу после выбора соревнования:
- **Транспондер** — работоспособность (✓/✗): false по умолчанию, true после первого проезда
- **Номер участника** — из таблицы Competitor
- **ФИО** — из таблицы Competitor
- **Модель** — название из RCModel
- **Масштаб** — из RCModel
- **Количество кругов** — из CompetitionLaps
- **Лучшее время круга** — из CompetitionLaps

## Структура проекта

```
trackpulse/
├── cmd/main.go              # Точка входа, DI контейнер
├── internal/
│   ├── config/              # Конфигурация
│   ├── database/            # БД
│   ├── locale/              # Переводы
│   ├── models/              # Сущности
│   ├── repository/          # CRUD
│   ├── service/             # Бизнес-логика
│   └── ui/                  # Fyne UI
├── pkg/                     # Публичные пакеты
└── tests/                   # Тесты
```

## Ключевые технологии

- **Go 1.19**
- **Fyne v2.7.3** — GUI фреймворк
- **SQLite3** — база данных
- **UUID** — идентификаторы
- **RFID** — сканирование кругов
- **go.bug.st/serial** — работа с последовательными портами
